# TraySubmission

클라이언트의 트레이 제출 요청을 마스터가 검증하고, 요청별 응답을 원래 제출자에게 반환하는 권위 기반 처리 코드입니다.

## 파일

| 파일 | 역할 |
|---|---|
| `TraySubmissionHandler.cs` | 요청 시퀀스, 응답 타임아웃, 마스터 검증, 승인·거절 콜백을 관리합니다. |
| `StageFlowRpcHandler.cs` | 스테이지 공용 RPC 허브입니다. 포트폴리오에서는 트레이 요청·응답 및 ACK 관련 구간이 핵심입니다. |

## 제출 흐름

```text
클라이언트가 Tray ViewID와 sequenceId 전송
→ 마스터가 RPC 발신자 ActorNumber 확인
→ 마스터의 실제 TrayItem 상태 조회
→ 소유자와 제출자 일치 여부 검증
→ 권위 있는 SubmittedTray 스냅샷 생성
→ 승인 또는 거절 응답
→ 요청자는 sequenceId가 일치할 때만 콜백 처리
```

## 설계 포인트

- 클라이언트가 계산한 레시피 결과를 신뢰하지 않고, 마스터가 현재 네트워크 오브젝트 상태를 다시 읽습니다.
- `sequenceId`로 지연된 이전 응답이 새로운 요청을 완료시키는 것을 방지합니다.
- 응답이 없으면 제한 시간 후 대기 상태를 정리해 재요청할 수 있도록 복구합니다.
