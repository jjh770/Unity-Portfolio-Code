# Unity Portfolio Code Samples

유니티 클라이언트 개발 프로젝트에서 직접 담당한 핵심 코드만 선별한 포트폴리오 저장소입니다.
전체 Unity 프로젝트를 배포하는 저장소가 아니라, **문제 해결 과정과 설계 판단을 코드 단위로 검토할 수 있도록 구성한 샘플 저장소**입니다.

## 프로젝트

| 프로젝트 | 핵심 주제 | 대표 코드 |
|---|---|---|
| [Don’t Dilly-Dally!](./DontDillyDally/README.md) | Photon PUN2 기반 권한·상태 동기화와 스테이지 초기화 | ACK 재동기화, 아이템 소유권, 권위 기반 제출 판정 |
| [Beat & Buddy](./Beat%26Buddy/README.md) | 오디오 시간축 기반 리듬 시스템과 채보 데이터 파이프라인 | `dspTime`, Beat 기반 노트 이동, `.osu` 파싱 |
| [Flora333](./Flora333/README.md) | 기능 확장을 고려한 플레이어 컴포넌트 구조 | FSM, 입력 이벤트 분리, 상호작용·스킬 시스템 |
| [Monster Kindergarten](./MonsterKindergarten/README.md) | 저장소 추상화와 도메인 중심 게임 데이터 관리 | Hybrid Repository, 도메인 규칙, 클릭·머지 루프 |

## 저장소 구성 원칙

- 본인이 담당한 기능의 핵심 코드만 선별했습니다.
- 팀 프로젝트의 다른 담당자 코드, 외부 SDK 소스, 유료 에셋은 포함하지 않았습니다.
- 씬, 프리팹, ScriptableObject 에셋과 일부 공통 타입이 제외되어 **단독 실행용 프로젝트는 아닙니다.**
- 각 프로젝트와 기능 폴더의 README에서 파일 관계와 권장 읽기 순서를 설명합니다.
- 코드 주석은 문법을 설명하기보다, 네트워크 권한·타이밍·취소·복구 정책처럼 코드만으로 알기 어려운 설계 의도를 중심으로 작성했습니다.

## 권장 검토 순서

1. 프로젝트 README에서 문제와 설계 목표 확인
2. 기능 폴더 README에서 실행 흐름 확인
3. README에 표시된 핵심 파일 순서대로 코드 검토
4. 메서드 내부의 설계 의도 주석 확인

## 참고

이 저장소는 포트폴리오 검토를 위한 코드 샘플입니다. 실제 게임에는 UI, 사운드, 연출, 씬 구성, 데이터 에셋과 추가 시스템이 함께 연결되어 있습니다.
